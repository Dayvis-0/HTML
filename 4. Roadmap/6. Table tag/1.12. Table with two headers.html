Table with two headers

Example 1: Table with header cells in the top row and first column

The following table of apening times has header information in both the top row and the first column. All header cells are marked up as <th> cells with scope attributes added.

In the header row, the col value for scope associates each header cell with the data cells in the column. In the header column, the row value associates the individual headers with their rows. Without this information, some users would not easly understand the relationship between header and data cells. In the example below, the "Open" and "Closed" labels are only useful if they can be assigned to a particular day and time.

Example 2: Table with an offset column of header cells

In this table, the row header cells are in the second rather than in the first column. The approach is similar to the examples above: The scope of the header cells in the top row is set to col. By using the row value for scope assigns the header cell in the second column to the data cells on the left and the right of the individual header cell.
